# Safe Artifact Cleanup Design (No Runtime Breakage)

## Objective
Clean generated cache/artifact files from the project while preserving current runtime behavior and keeping `venv/` intact.

## Scope
In scope:
- Remove generated Python cache artifacts from working tree.
- Ensure generated cache artifacts stay ignored by git.
- Keep current execution flow working normally.

Out of scope:
- Deleting `venv/`.
- Refactoring business logic.
- Removing functional source files.

## Constraints (User-confirmed)
- Only remove items that do not break current flow.
- Keep virtual environment (`venv/`) as-is.

## Cleanup Target
Primary targets:
- `**/__pycache__/`
- `**/*.pyc`
- `.pytest_cache/`

Non-targets for this cleanup pass:
- `venv/`
- Active source/test files
- Core config/runtime files

## Approach
1. Verify `.gitignore` includes robust rules for Python cache artifacts.
2. Remove cache artifacts from filesystem (safe generated files only).
3. Untrack any cache artifacts still in git index (if present).
4. Run lightweight verification (`git status`, selected tests) to confirm no runtime-impacting change.

## Git Ignore Policy
Expected ignore rules:
- `__pycache__/`
- `*.py[cod]`
- `.pytest_cache/`

These prevent future noise from cache generation during normal runs/tests.

## Safety Guarantees
- No deletion of runtime dependencies in `venv/`.
- No deletion of source code (`.py`) except generated bytecode (`.pyc`).
- No behavioral code changes in pipeline logic.

## Verification Plan
After cleanup:
1. `git status` should no longer show cache artifact churn.
2. Run fast test subset (existing core tests already used in this project flow).
3. Confirm CLI still starts normally.

## Acceptance Criteria
- Cache artifacts removed from working tree.
- Cache artifacts are ignored by git moving forward.
- `venv/` remains untouched.
- No regressions in basic test run.
